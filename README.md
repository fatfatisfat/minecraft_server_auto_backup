因為自動備份的模組不支援新版本的伺服器，在查資料的時候偶然發現可以自己寫一個自動化腳本來達到我需要的效果
使用語言: Batch

關鍵改動:

1. 路徑與編碼問題

    。遇到的問題：<br>
        最初使用中文資料夾名稱「中華帝國」，導致腳本在執行 tar 或路徑跳轉時出現亂碼，程式找不到目標<br>
        腳本啟動後立即閃退，或顯示 tar: (null) 與 Couldn't visit directory<br>

    。解決方法：<br>
        將地圖資料夾名稱從中文改為英文<br>
        將 .bat 檔案的儲存編碼從 UTF-8 改為 ANSI<br>

2. 檔案鎖定衝突

    。遇到的問題：<br>
        在伺服器運作時執行備份，會出現 PermissionDenied 錯誤，提示 session.lock 檔案正在被其他處理序（Minecraft 伺服器）使用
        這導致壓縮程式（PowerShell 或 tar）直接中斷，產出的備份檔僅有 20 KB 的空殼<br>

    。解決方法：<br>
        在壓縮前，先使用 robocopy 指令將地圖資料夾同步到 temp_backup 暫存區<br>
        設定 robocopy 參數使其遇到被鎖定的檔案（如 session.lock）時直接跳過而不報錯，確保備份過程不中斷<br>

3. 自動偵測與空間管理

    。遇到的問題：<br>
        手動輸入資料夾名稱容易出錯且不夠彈性<br>
        擔心持續備份會塞爆儲存空間 <br>

    。解決方法：<br>
        空間檢查：利用 PowerShell 指令偵測剩餘空間，低於 5 GB 則自動跳過備份以保護系統<br>
        智慧清理策略：將資料進行稀疏化處理，保留當天所有備份，隔天以後每天僅留一份，並自動刪除超過 7 天的舊檔<br> 
        ![備份和存檔策略](images/backup-strategy.png)<br>

4. 效能優化：事件驅動

    。遇到的問題：<br>
        即便沒人線上，伺服器依然每 10 分鐘備份一次，造成硬碟空間與 CPU 資源的浪費<br>

    。解決方法：<br>
        連線偵測：使用 netstat 指令檢查伺服器連接埠是否有建立連線<br>
        設定「有玩家才備份」邏輯，偵測到玩家連線時才啟動備份流程，否則直接進入下一個週期的等待<br>

5. 新增運行進度條

    。遇到的問題：<br>
        終端機的畫面看起來像是當機一樣卡住，我無法一眼確認是否為故障
    
    。解決方法：<br>
        加上進度條，提供可視化的運作情況
